## Lançamento de Projeto
Como uma atividade de Descanso (veja Descanso no Capítulo 1: O Básico), você pode fazer um Lançamento de Projeto para um de seus projetos. Outra opção é fazer um Lançamento de Projeto para contribuir com o projeto de outro herói. Um Lançamento de Projeto é um Teste com um resultado especial que não é dividido em Graus. A Característica usada no Teste é determinada pelo projeto.

Ao fazer um Lançamento de Projeto, o total - não importa o quão baixo - se torna uma quantia em Pontos de Projeto que é acumulada para o projeto. Mesmo que tenha uma penalidade em um Lançamento de Projeto, o total mínimo para o resultado é 1. Se você tiver um Sucesso Crítico - um 19 ou 20 Natural - em um Lançamento de Projeto, isto é chamado Avanço e você pode fazer outro Lançamento de Projeto para o mesmo projeto como parte da mesma atividade de Descanso.

Um Projeto de Tempo Livre é concluído quando acumular uma quantidade de Pontos de Projeto igual ou maior que o Objetivo de Projeto - o número de pontos que é necessário para concluir o projeto, contido na descrição de cada projeto.

Conforme você avança em um Projeto de Tempo Livre, o Diretor pode adicionar eventos que ajudam a formar a narrativa da sua pesquisa, criação e outras atividades. Esses eventos podem fornecer benefícios ou desafios inesperados ao longo do seu projeto, para tornar a jornada tão dramática quanto o destino. Veja Para o Diretor: Eventos de Projeto a seguir para mais informações.

### Vantagens e Desvantagens em Lançamentos de Projeto
Uma vez que Lançamentos de Projeto não usam Graus de Sucesso, eles não seguem as regras usuais para Vantagens e Desvantagens Duplas. Se você ganhar uma Vantagem em um Lançamento de Projeto, some 2 ao resultado, como sempre. Mas se tiver uma Vantagem Dupla, some 4 ao resultado. De igual maneira, subtraia 2 de um Lançamento de Projeto que tiver uma Desvantagem e subtraia 4 de um resultado que tenha uma Desvantagem Dupla.

### Perícias e Lançamentos de Projeto
Você pode aplicar Perícias dos Grupos de Perícias de Criação ou Conhecimento em Lançamentos de Projeto que sejam diretamente relacionados a um Projeto de Tempo Livre (veja Perícias em Capítulo 9: Testes). Por exemplo, você pode usar a Perícia Alfaiataria para Lançamentos de Projeto relacionados à criação de um casaco magico e pode usar a Perícia História ao tentar pesquisar a localização de uma antiga batalha onde uma coroa mágica foi perdida. A critério do Diretor, você também pode usar Perícias de outros Grupos de Perícias, mas uma Perícia usada em um Lançamento de Projeto deve ser diretamente relacionada ao projeto.

### Idioma e Lançamentos de Projeto
Se você souber o Idioma de uma Origem do Projeto, você pode fazer um Lançamento de Projeto sem problemas. Se não souber o Idioma da Origem do Projeto, mas souber um Idioma relacionado, o Lançamento de Projeto tem uma Desvantagem. Se não souber o Idioma da Origem do Projeto nem um Idioma relacionado, o Lançamento de Projeto tem uma Desvantagem Dupla.

### Guias
Heróis às vezes encontram Guias - livros especiais, esquemas, NPCs inteligentes, registros sobrenaturais, etc. - que fornecem informações importantes e fáceis de entender a respeito de um Projeto de Tempo Livre. Cada Guia tem um valor de conhecimento e uma conexão a um projeto específico que reduz significativamente o tempo necessário para concluir o projeto. Quando um herói estuda um Guia como uma atividade de Descanso, ele ganha o valor de conhecimento do Guia como Pontos de Projeto para o projeto. O Guia não pode ser usado novamente naquele caso específico de projeto, embora possa ser usado para um projeto diferente do mesmo tipo.

Guias são concedidos a critério do Diretor, geralmente como uma ferramenta que pode ser usada para aumentar a velocidade de projetos complexos para campanhas com pouco Tempo Livre. Por exemplo, o Projeto de Tempo Livre Construir um Dirigível tem um Objetivo de Projeto de 3.000, o que requer uma quantidade considerável de Tempo Livre. Mas se um herói encontrar um manual que descreve como construir um dirigível e concede 1.000 ou mais Pontos de Projeto para o projeto Construir um Dirigível, o tempo de criação pode ser reduzido significativamente.

Um Guia deve fornecer sua informação em um Idioma que você entende para receber todo o benefício. Se um Guia usar um Idioma relacionado a algum Idioma que você conhece, você ganha apenas metade do valor de conhecimento do Guia (arredondado para baixo) como Pontos de Projeto para o projeto.

### Para o Diretor: Eventos de Projeto
Eventos de Projeto são eventos narrativos que apresentam benefícios e desafios para heróis conforme completam projetos de pequisa e criação. Esses eventos são totalmente opcionais. Se seu grupo prefere um jogo onde os heróis simplesmente avançam em seus objetivos com Lançamentos de Projeto, tudo bem. Mas usar estes eventos pode ajudar o Diretor a injetar mais drama nos processos de pesquisa e criação.

#### Quando Usar Eventos de Projeto
Projetos não precisam de um evento sempre que um herói fizer um Lançamento de Projeto. Se fosse o caso, os heróis provavelmente usariam todo o seu tempo na organização de projetos e nunca iriam em aventuras. Em vez disso, você, como Diretor, escolhe um dos métodos a seguir para determinar quando um evento ocorre durante um Projeto de Tempo Livre. Você sempre pode trocar o método usado durante uma campanha e escolher fazer o que funcionar melhor para a situação atual.

#### Lançar para Evento
Se quiser que eventos sejam uma surpresa para você tanto quanto para os outros jogadores, então sempre que um ou mais heróis fizerem um Lançamento de Projeto durante um Descanso, jogue um d6. Se o resultado for 6, um evento acontece. Esta é uma boa opção se você gosta de improvisar a história durante o jogo.

#### Marcos de Evento
Se você quiser garantir que eventos ocorram durante um Projeto de Tempo Livre, use Marcos de Evento baseados em Pontos de Projeto. Quando o projeto acumula um certo número de pontos, um evento ocorre na próxima vez que um herói quiser fazer um Lançamento de Projeto. Esta abordagem funciona bem se você quiser planejar seu evento em detalhes antes que ocorra.

Você pode usar a tabela de Marcos de Evento Sugeridos para determinar quando um evento deve ocorrer durante um projeto.

##### Marcos de Evento Sugeridos

| Objetivo de Projeto | Marcos                                                                     |
| ------------------- | -------------------------------------------------------------------------- |
| 30 ou menos         | Nenhum                                                                     |
| 31 - 200            | Um evento na metade do caminho até o objetivo                              |
| 201 - 999           | Dois eventos em um terço e dois terços do caminho até o objetivo           |
| 1.000 ou mais       | Três eventos em um quarto, metade e três quartos do caminho até o objetivo |

#### Quando Quiser
Sempre que pensar que os heróis precisam de um pouco de drama durante um Descanso, use um evento. Esta abordagem permite planejar um evento detalhadamente quando um Projeto de Tempo Livre começar e então usar o evento no momento mais dramático - ou até durante um encontro se você se sentir ousado!

#### Aplicando Eventos
Um evento ocorre quando um herói faz um Lançamento de Projeto. Você pode jogar dados ou escolher um evento a partir da tabela de sugestões adequada, ou usar as tabelas como inspiração para seus próprios eventos. Cada evento inserido na tabela é uma sugestão narrativa escrita para você, como Diretor, e que você pode modificar ou detalhar como precisar. Cada evento especifica se ocorre antes ou depois do Lançamento de Projeto.

#### Avanço Automático
Se um evento concede um Avanço automático em um Projeto de Tempo Livre, o projeto ganha 20 Pontos de Projeto e o herói pode fazer outro Lançamento de Projeto para o mesmo projeto como parte da mesma atividade de Descanso.
